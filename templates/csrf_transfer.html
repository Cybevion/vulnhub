💡How CSRF Works
1. Victim logs into bank.com → Gets session cookie: session=abc123 2. Victim visits evil.com → evil.com has a hidden auto-submit form → Target: bank.com/transfer 3. Browser sends POST to bank.com → Automatically includes session=abc123 → Bank sees authenticated request 4. Transfer happens without victim knowing
Fix: CSRF tokens — random secret embedded in form. Attacker cannot read it cross-origin (SOP). Also: SameSite=Strict cookie attribute blocks this entirely in modern browsers.
Note: XSS defeats CSRF tokens — if attacker can run JS on your domain, they can read the token. Fix XSS first.
